# Knovia Prototype Demo Operation Guide

This guide explains exactly how to operate the prototype to showcase all major product features.

## 1) Launch

1. Run `npm install` (first time only).
2. Run `npm run dev`.
3. Open the local Vite URL in browser.

## 2) Demo Goal

Show that Knovia provides a runnable learning process for a Data Analyst job sprint:

- package selection at session creation
- creator-side package generation and publishing flow
- configurable intake page driven by package creator requirements
- personalized planning from learner-provided materials
- graph-based skill progression
- multi-session branching (Ask/Explain)
- rich artifact canvas
- mastery-style node progression

## 3) Recommended Live Demo Script (8-10 minutes)

### Final Showcase Sequence (12-15 min)

- 0:00-3:00 Creator Studio: create skill packs + commands + intake, then publish.
- 3:00-6:00 Launch published package from Welcome -> setup intake -> generate plan.
- 6:00-9:00 Enter runtime -> open topic -> branch Ask/Explain sessions.
- 9:00-11:00 Type `/` -> choose command from popover -> fill command dialog -> run.
- 11:00-13:00 Show command execution card and content artifacts generated by package logic.

### Step 0 - Creator Studio (2 min)

1. On Welcome page, click **Open Creator Studio**.
2. Upload a source material file.
3. Define **Agent Skill Packs** and **Commands** in the editor.
4. Configure learner intake schema and runtime policy.
5. Click **Publish Runnable Skill Package**.
5. Return to Welcome page and show the new package appears in **Course Packages**.

Feature shown:
- creator workflow from raw material to runnable skill package
- package schema-driven publishing pipeline (skills + commands + intake + policy)

### Step A - Start from Setup (1.5 min)

1. On Welcome page, click **Data Analyst Job Sprint** in the **Course packages** section.
3. On setup page, highlight **Creator guidance** (this defines what materials learners must provide).
4. Click **Fill Sample Inputs** (or manually upload/input your own materials).
5. Point out all provided fields (Resume, GitHub URL, optional target role).
6. Click **Generate Learning Plan**.

Feature shown:
- package-specific, creator-configurable intake requirements
- multi-input learner context collection
- personalized onboarding flow

Optional 30-second add-on:
- Return to Welcome page and click **Product Analyst Growth Sprint**.
- Show that intake requirements change automatically (Product Brief + Dashboard URL + Focus Question).

### Step B - Planning Flow (2 min)

1. In chat, click **Continue** repeatedly until all planning steps complete.
2. Explain that the system is generating a path for "Data Analyst Job Sprint".
3. Click **Start Learning**.

Feature shown:
- Plan generation workflow
- transparent step-by-step planning

### Step C - Skill Graph Runtime (2 min)

1. In Skill Graph, click **SQL Foundations** to open topic session.
2. Point to locked vs ready vs in-progress states in the left skill list.
3. Switch between **Skill Tree** and **Content** tabs.

Feature shown:
- Graph/Plan/State runtime
- dependency-based progression

### Step D - Multi-Session Branching (2 min)

1. In chat, ask a question (for example: "How do I avoid SQL mistakes in interviews?").
2. Select part of assistant response text.
3. Click **Ask** in the popover to create a branch session.
4. Repeat with **Explain** for another branch.
5. Click branch nodes on the graph to navigate across sessions.

Feature shown:
- one node, multiple sessions
- non-linear learning branches

### Step D2 - Command Palette and Execution (1.5 min)

1. In chat input, type `/`.
2. Show the command popover with prefix filtering.
3. Click one command to open argument dialog.
4. Fill inputs and click **Run Command**.
5. Show system message "Executing command ..." and resulting response.

Feature shown:
- user-friendly command UX (no technical syntax required)
- explicit command execution feedback
- creator IP protection (runtime uses package logic without exposing internal skill instructions)

### Step E - Canvas Artifacts (2 min)

1. In topic or branch chat, click quick actions such as:
   - **Skill Dependency Map**
   - **SQL vs BI Focus Matrix**
   - **Interview Timeline**
   - **Run mastery check**
2. Show generated artifacts in Content tab.
3. Select text inside canvas and branch again with Ask/Explain.

Feature shown:
- AI as New UI (artifact-first learning canvas)
- reusable structured learning outputs

### Step F - Mastery Progress Signal (0.5 min)

1. Click **Mark Node Mastered** on a topic session.
2. Return to skill graph and show status update + next node unlock.

Feature shown:
- progress tied to node mastery state, not linear video consumption

## 4) Backup Prompts (if live interaction stalls)

Use these exact prompts in chat:

- "Show me a skill dependency map for interview readiness"
- "Give me a SQL vs dashboard tradeoff comparison"
- "Run a funnel diagnosis walkthrough"
- "Give me interview anti-patterns"
- "Create a case answer blueprint"

## 5) Troubleshooting During Demo

- If no rich content appears: use explicit pack command format `@content:<pack-id>`.
  - Example: `@content:hooks-lifecycle-map`
- If branch popover does not appear:
  - select a longer text snippet (3+ characters)
  - release mouse inside chat/content panel
- If session graph looks crowded:
  - click the desired skill on the left list to focus on that node tree

## 7) Suggested Talking Points (One-liners)

- "This is not a static course player. It is a runnable learning runtime."
- "Each package can define its own intake requirements, so onboarding is configurable by creators."
- "New session creation supports package selection, and each package renders its own intake schema."
- "Each skill node can branch into multiple sessions, so learning is not trapped in one linear chat."
- "The canvas stores reusable learning artifacts: maps, timelines, checklists, and case blueprints."
- "Progress is tracked on node mastery and dependency unlocks, not passive watch-time."
